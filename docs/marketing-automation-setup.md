# Marketing Automation Setup

This module creates AI-assisted social campaigns for Instagram, Facebook, and WhatsApp, but it never auto-posts without owner/admin approval.

For social-to-website lead capture, product recommendation links, Meta webhook setup, and n8n workflow configuration, see `docs/omnichannel-commerce-configuration.md`.

## What this module needs

### 1. OpenAI
Used for caption, hashtag, CTA, and image prompt generation.

Required:
- `MARKETING_AI_ENABLED=true`
- `MARKETING_OPENAI_API_KEY`
- optional: `MARKETING_OPENAI_MODEL`
- optional: `MARKETING_IMAGE_PROVIDER=LEONARDO`

If the API key is missing, the app falls back to safe mock AI drafts.

### 2. Leonardo AI
Used for campaign image generation when `MARKETING_IMAGE_PROVIDER=LEONARDO`.

Required:
- `MARKETING_LEONARDO_API_KEY`

Recommended defaults:
- `MARKETING_LEONARDO_MODEL_ID=de7d3faf-762f-48e0-b3b7-9d0ac3a3fcf3`
- `MARKETING_LEONARDO_STYLE_UUID=111dc692-d470-4eec-b791-3475abac4c46`

Optional tuning:
- `MARKETING_LEONARDO_WIDTH`
- `MARKETING_LEONARDO_HEIGHT`
- `MARKETING_LEONARDO_CONTRAST`
- `MARKETING_LEONARDO_POLL_ATTEMPTS`
- `MARKETING_LEONARDO_POLL_DELAY_MS`

If Leonardo is missing or fails, the app falls back to a built-in creative preview so the approval workflow keeps moving.

### 3. Meta (Instagram + Facebook)
Used for real publishing to Instagram and Facebook through the Meta Graph API.

Required:
- `MARKETING_META_ACCESS_TOKEN`
- `MARKETING_INSTAGRAM_BUSINESS_ACCOUNT_ID`
- `MARKETING_FACEBOOK_PAGE_ID`
- optional: `MARKETING_META_GRAPH_VERSION`

Without these values, Instagram/Facebook publishing stays in placeholder mode and will fail with a clear log message.

### 4. WhatsApp Provider
The module supports a provider switch:
- `MARKETING_WHATSAPP_PROVIDER=GUPSHUP`
- or `MARKETING_WHATSAPP_PROVIDER=TWILIO`

## Gupshup setup

For Gupshup marketing publishing, the app needs:
- `MARKETING_GUPSHUP_API_KEY`
- `MARKETING_GUPSHUP_APP_NAME`
- `MARKETING_GUPSHUP_SOURCE_NUMBER`
- `MARKETING_GUPSHUP_TEMPLATE_ID`

Current WhatsApp message template is designed around these placeholders:
- `{{1}}` customer name
- `{{2}}` discount value
- `{{3}}` product / offer subject
- `{{4}}` short body line generated by AI
- `{{5}}` landing URL
- `{{6}}` offer valid-until date

### Important
At the moment, the missing value is:
- `MARKETING_GUPSHUP_SOURCE_NUMBER`

You also need the final approved template id from Gupshup for the exact promotional template that will be used in production.

## Twilio setup

If you ever switch back to Twilio for WhatsApp marketing:
- `MARKETING_TWILIO_ACCOUNT_SID`
- `MARKETING_TWILIO_AUTH_TOKEN`
- `MARKETING_TWILIO_WHATSAPP_FROM`
- optional: `MARKETING_TWILIO_OFFER_CONTENT_SID`

## Approval roles

- `ADMIN` can create, edit, and generate campaigns
- `OWNER` can approve, reject, schedule, and publish
- staff access is controlled through the `MARKETING_AUTOMATION` permission

## Seed data

The local seed includes one example campaign:
- `Akshaya Tritiya Jewellery Glow`

It creates pending-approval draft content for:
- Instagram
- Facebook
- WhatsApp

## Publishing behavior

- AI generation creates drafts only
- drafts move into `PENDING_APPROVAL`
- only `APPROVED` content can be scheduled or published
- scheduler checks due `SCHEDULED` content every minute
- every publish attempt is stored in `publish_logs`
- analytics rows are created for published content and can be extended with real platform metrics later
