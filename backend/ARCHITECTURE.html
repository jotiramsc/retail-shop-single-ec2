Retail Shop Backend - Architecture

Spring Boot 3.3.5 | Java 17 | PostgreSQL | Redis | Omnichannel + WhatsApp AI

Controllers: REST entrypoints
Services: domain/business logic
Repositories: JPA persistence
Integrations: Razorpay, Meta, Gupshup, OpenAI, AWS

1) Client / Channel Layer

Web Storefront
Catalog, cart, checkout, customer account.
Admin UI
Products, billing, campaigns, reports, staff controls.
WhatsApp
Template messages, bot conversations, order notifications.
Meta / Social
Lead capture and webhook event intake.

2) API + Security Layer

Controllers
/api/products, /api/order, /api/auth, /api/whatsapp/webhook.
src/main/java/com/retailshop/controller
JWT Security
Separate staff/customer filters and token services.
SecurityConfig, StaffJwtFilter, CustomerJwtFilter
RBAC
Role + permission authority checks via @PreAuthorize.
Error Handling
Centralized API errors in GlobalExceptionHandler.

3) Domain Services Layer

Commerce
Product, Cart, Checkout, Order, Offer, Billing services.
Customer Auth
OTP + Google login + profile completion lifecycle.
Payments
Razorpay order creation, verification, webhook audit logs.
Omnichannel
Lead capture, product search cards, webhook signature validation.
Marketing
Campaign workflow, approvals, scheduling, publisher integrations.
WhatsApp Bot
Intent classification, memory retrieval, handoff orchestration.

4) Data + State Layer

PostgreSQL
Primary transactional store (products, orders, campaigns, leads).
src/main/resources/schema.sql
Redis (optional)
OTP/cart state acceleration behind feature flag app.redis.enabled.
RedisStateStore
JPA Repositories
Entity persistence and custom analytics queries.
src/main/java/com/retailshop/repository
Seed Data
SQL seed scripts and local profile seeder.
data.sql, data-local.sql, LocalDataSeeder

5) External Integration Layer

Payments
Razorpay APIs + webhook endpoint /api/razorpay/webhook.
Messaging
Meta WhatsApp or Gupshup provider switch via config.
AI
OpenAI for marketing + bot, Qdrant for semantic memory.
AWS
S3 storage, optional Secrets Manager token refresh persistence.

Key Runtime Flows

Checkout: Cart → Pricing → Payment Order → Verify Signature → Create Order + Invoice.

WhatsApp Bot: Webhook → Parse inbound → Classify intent → Fetch context/memory → Compose + send response.

Marketing Automation: Campaign content → Approval → Scheduler tick → Publisher (Meta/WhatsApp) → Analytics log.

Omnichannel Lead: Social webhook → Signature validate → Lead + conversation upsert → Product recommendations.
